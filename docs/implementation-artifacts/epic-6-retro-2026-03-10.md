## What Didn't Go Well

### 1. AC gaps in 6.2 — 3 HIGH findings in code review

Story 6.2 had three high-severity logic gaps that passed the initial implementation:
- First collection page load never transitioned to loading state after 1s (AC1)
- Memory log double-counted skeleton bytes in cache number (AC6)
- Status bar `last_warning` text only drew from session log, not engine warnings (AC5 partial)

These are not clippy-catchable issues — they are semantic gaps requiring AC-by-AC review before
submission.

### 2. Action item #1 from Epic 5 not anchored

Epic 5 retro action item #1 (integrate manual validation ACs in 6.2 for truncated/corrupted
files) was documented but not applied when the story was created. The SM did not carry it
through to the story spec.

### 3. Corrupted/truncated file validation never done on real data

AC3 and AC4 of 6.2 (incomplete file indicator, non-fatal navigation warnings) were validated
with artificially forced values only. Manual validation with a truncated `heapdump-visualvm.hprof`
was done informally. No test asset exists for repeatable validation.
