## What Didn't Go Well

### 1. No validation on a real large dump
LRU eviction was only exercised in unit tests with artificially forced budgets
(1 byte). The real scenario — navigating a 500 MB+ dump with a realistic budget
— was never validated. For an epic entirely about memory management, this is a
significant blind spot.

**Context:** `assets/` is gitignored (sensitive system data). `heapdump-rustrover.hprof`
is available locally and is the correct tool for this validation.

### 2. Post-review fix pattern persisting
Story 5.3 required 5 corrections after code review (unused import, missing
`Default`, absent `debug_assert`, incorrect constant imports, LRU order
verification). These are preventable with a pre-review local `cargo clippy --
-D warnings` pass. Clippy runs in CI but not consistently before submitting
for review.

### 3. Retro action item follow-through not visible
Epic 4 action item "update code review reports after corrections" had no
visible evidence of completion in Epic 5. The process isn't anchored.
